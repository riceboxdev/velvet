# REST API

**Base URL**: `https://velvetapi.com`

## Authentication

Velvet uses two authentication methods:

| Method | Use Case | Header |
|--------|----------|--------|
| **JWT Token** | User dashboard access | `Authorization: Bearer <token>` |
| **API Key** | Admin/programmatic access | `X-API-Key: wl_xxxxx` |

## Quick Reference

### Public Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/signup` | Join a waitlist |
| GET | `/api/signup/:referralCode` | Check signup status |
| GET | `/api/signup/check/:waitlistId/:email` | Check if registered |
| GET | `/api/waitlist/:waitlistId` | Get waitlist info |
| GET | `/api/waitlist/:waitlistId/leaderboard` | Get top referrers |

### User Dashboard Endpoints

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/signup` | Register new user |
| POST | `/api/auth/login` | Login |
| GET | `/api/auth/me` | Get current user |
| GET | `/api/user/waitlists` | List user's waitlists |
| POST | `/api/user/waitlists` | Create waitlist |

### Admin API (API Key)

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/admin/waitlist` | Get waitlist with stats |
| GET | `/api/admin/signups` | List all signups |
| PATCH | `/api/admin/signups/:id/offboard` | Admit user |
| DELETE | `/api/admin/signups/:id` | Remove signup |
