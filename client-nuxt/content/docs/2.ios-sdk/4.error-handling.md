# Error Handling

The SDK throws `WaitlistError` for various error conditions.

## Error Types

| Error | Description |
|-------|-------------|
| `invalidConfiguration` | API key or waitlist ID is missing/invalid |
| `invalidEmail` | Email address failed validation |
| `alreadySignedUp` | Email is already on the waitlist |
| `networkError(Error)` | Underlying network error |
| `serverError(String)` | Backend returned an error |
| `unknown` | Unexpected error occurred |

## Example

```swift
do {
    let response = try await client.signup(email: "user@example.com")
} catch let error as WaitlistError {
    switch error {
    case .invalidConfiguration:
        print("Check your API key and waitlist ID")
    case .invalidEmail:
        print("Please enter a valid email")
    case .alreadySignedUp:
        print("You've already joined!")
    case .networkError(let underlyingError):
        print("Network error: \(underlyingError)")
    case .serverError(let message):
        print("Server error: \(message)")
    case .unknown:
        print("Something went wrong")
    }
}
```

## Troubleshooting

### "Invalid Configuration" Error

- Verify your API key and waitlist ID are correct
- Check that the base URL includes the protocol (`https://`)

### Network Errors

- Verify device has internet connectivity
- Check that the backend URL is accessible

### Users Not Seeing Confirmation

- The SDK stores state locally per device
- Reset simulator to clear stored state when testing
